# Feynman "mastered" list - <project name>

This is the memory for the `feynman-explanation` skill's training-wheels protocol: the running
list of symbols, acronyms, and concepts the learner has explicitly said they own. Once an item
is here, the skill STOPS defining or re-glossing it.

It is per-project and personal. Copy this template into the project you are working in (for
example `docs/feynman-mastered.md`) and gitignore it if that project is public. The skill reads
it at the start of explanation work and appends to it the moment the learner confirms a new item.

Training wheels come off PER ITEM, not all at once: owning `=>` does not mean owning `async`.

---

## How to add an item

When the learner says they own something ("I've got `=>`", "I know what CI is"), add one line
below, newest first, in the same turn while it is fresh:

```markdown
- `<item>` - <one-line note on what they own> (confirmed <date>)
```
